# 2026-06-12 vault-memory-recall

BUILT + LIVE-DEMOED: vault-backed retrieval memory — **recall instead of re-read**. `triad/memory.py` (`VaultMemory`: lexical TF-IDF index over every `.md` in the vault; `rank`, `recall`, `savings`, `stats`) + REPL `/recall <query>` (needs `--vault`). **143/143 offline tests** (+9 memory). **LIVE on this vault** (24 notes, 112 chunks, ~7247 tokens): per-query recall pulls the right note(s) at **96–98% fewer tokens** than dumping the whole vault — e.g. "how do three free models avoid correlated errors" → [[cross-provider decorrelation]] in ~86 tokens vs ~7247.

**Why this matters (the target — see project memory):** the audience isn't only $0 users; it's **paying users on capped tiers** who hit context-window / token / rate limits. Agents normally re-send the whole growing transcript every turn (cost scales with conversation length). Persisting to the vault (vault.py already does) + retrieving only the relevant slice = **long memory on a small context window**. Same principle as the [[protocol]] handoff ("reference, don't re-quote"), now across sessions and addressable by relevance.

**Design choice (deliberate, budget-driven):** retrieval is **lexical (TF-IDF), not embeddings**. Embeddings would cost an API call / token spend per turn — defeats the goal. Words-in-common is free, offline, no new deps, and good enough to surface notes from a vault you wrote yourself. Honest limit: lexical can mis-rank when many notes share vocabulary (one demo query surfaced a dense scoped note over the topic note) — acceptable for v1; an optional embedding tier is a future opt-in.

**AUTO-INJECTION NOW WIRED (loop closed):** parallel mode recalls relevant vault notes each turn and injects them as **ephemeral system context** (`Agent.stream(user_msg, context=...)` — prepended for that call only, never stored in history, so recall doesn't compound), and `Orchestrator.history_limit` caps per-agent history (`_trim_history`) so the re-sent context stops growing — older turns live in the vault, pulled back on demand. CLI `--memory` (+ `--history-limit N`, default cap 6 when `--memory`) and REPL `/memory on|off`; memory is rebuilt each turn so it sees notes earlier turns wrote. **148/148 offline tests** (+5 wiring: injection present, NOT stored in history, queried with the task, history bounded, keeps most-recent turn). **LIVE-verified**: `triad --free --vault Claude --memory` recalled ~383 tokens of relevant context and capped history on a real turn. Targets parallel (where persistent history grows); relay/council are ephemeral + already use the protocol handoff.

**Still open:** the **token-savings benchmark** (measure protocol + memory savings as a defensible number like the pass-rate bench); optional embedding-tier retrieval; summarize-on-evict (store a turn summary to the vault when trimming, so trimmed detail is still recallable).
